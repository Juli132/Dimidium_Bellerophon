Bellerophon Language Documentation

Language Overview

Bellerophon is a domain-specific scripting language (DSL) for 3D printing. Write clean, procedural motions and parametric macros that compile directly to G-code for Klipper or Marlin firmware.

Philosophy: Precise, readable commands that make controlling 3D printers easier.

Getting Started & Platform Requirements

Bellerophon is a 3D printer scripting IDE and language designed for Klipper and Marlin macros. It allows you to write structured, parametric scripts and compile them into firmware‑compatible G-code macros. This section describes system requirements, inputs, outputs, capabilities, and limitations.

1. Requirements

  • Software: Java Runtime Environment (JRE) 21 or newer
  • Hardware: Any computer capable of running Java

2. Running the IDE

Windows

  1. Double-click run.bat to start the server
  2. Your browser will open automatically to the Dimidium Ecosystem
  3. Close the server window when finished

Linux / Mac

  1. Install Java (OpenJDK 21 or newer)
  2. Open a terminal in the SyntaxN folder
  3. Run: java -jar SyntaxN.jar
  4. Open browser to: http://localhost:4567
  5. Press Ctrl+C in terminal to stop the server
The server runs locally on port 4567. No internet connection required after installation.

3. Compilation Targets: Klipper vs Marlin

The IDE can compile your .bph scripts to two different output formats:

  • Klipper – Full support for all Bellerophon features, including if statements, advanced temperature commands, and Klipper‑specific G‑code macros.
  • Marlin – A simpler output that omits features not available in Marlin firmware. Conditional statements (if/endif) are not supported in Marlin mode; all other movement, heating, looping, and macro commands work as expected (loops are expanded at compile time).

Select the target in the IDE before compiling. The generated .cfg (Klipper) or .gcode (Marlin) file will be saved accordingly.

4. Inputs

  • Bellerophon accepts only .bph script files.
  • The IDE provides real-time syntax validation, highlighting errors as you type.
  • Scripts are line-oriented: each command must be on its own line; blank lines between commands are ignored.
  • Supported commands include: movement, temperature control, heaters, fans, waits, pauses, macros calls, repeat loops, Brepeat loops, conditional execution (Klipper only), extruder control, and bed/probe operations.

5. Outputs

  • Compiles .bph scripts into either Klipper macro .cfg files or Marlin .gcode files.
  • Compiled macros can be deployed to printers via Gravity Hub or directly from Mainsail (Klipper) or any Marlin host.
  • The IDE provides Build Log and Exceptions tabs for debugging and catching compilation errors.

6. Capabilities

  • Compile structured macros for Klipper or Marlin with correct syntax.
  • Support for arithmetic expressions in coordinates and parameters: globally available
  • Mathematical functions: sin, cos, tan, sqrt, abs (absolute value), sign (signum), power operator ^, and constant pi.
  • Parametric loops using repeat (static repetition) and Brepeat (dynamic, iteration‑aware loops), including support for nested loops and iterator‑based expressions.
  • Parametric Geometry: Mathematical functions allow the generation of procedural motion paths such as circles, spirals, or arbitrary curves.
  • Conditional execution with if statements (Klipper only), allowing decisions based on temperatures or other parameters.
  • Temperature management for extruder, bed, and chamber (Set_Heater_Temperature, Heat, WaitForTemp).
  • Printer movement commands, including absolute and relative positioning, X/Y/Z moves, extruder control, and dwell/wait commands.
  • Macro calls within macros (M.call), allowing reuse and modular scripts.
  • Bed and probe management: BED_MESH_CALIBRATE, PROBE_CALIBRATE, LoadBedMesh.
  • Fan control, pressure advance, speed settings, and idle timeout (TIMEOUT_SET).
  • Pause, resume, cooldown, and SD card file printing (PRINTFILE).
  • IDE Reference Panel: Displays all Bellerophon tokens in a user-friendly format. Users can pin commands to a sidebar for quick access while typing, reducing errors and speeding up workflow.
  • Session Restoration: The IDE automatically restores the previous editing session to prevent accidental data loss and allow users to continue work seamlessly.
  • Dimidium Ecosystem Integration: Compiled macros can be deployed and managed through the Dimidium Gravity Hub dashboard or executed directly within Mainsail (Klipper) or any Marlin host.

7. Limitations

  • Cannot slice 3D models; only generates G-code from Bellerophon .bph scripts.
  • Does not modify existing G-code files automatically.
  • Some advanced or rare G-code commands may not be supported.
  • No general-purpose variables; logic is limited to loop iterators and math expressions
  • Marlin target: Conditional statements (if/endif) are not supported and will be ignored

9. Workflow Diagram

High-level workflow of using Bellerophon:

.bph script  →  IDE (edit & validate)  →  Select target → Compile →  .cfg (Klipper) / .gcode (Marlin) → Deploy → Run macros

Program Structure

A Bellerophon program consists of one or more macros. Each macro starts with M.title and ends with M.end.

M.title "Macro Name" [statement 1] [statement 2] [statement 3] M.end

Each command must be on its own line. Indentation is optional but recommended. Comments start with #.

Loops & Arithmetic Operators

Bellerophon supports two types of loops: repeat for static repetition and Brepeat for dynamic loops.

Positioning Context in Loops: The behavior of movement inside a loop depends heavily on the active mode:
  • Relative (G91): Coordinates are calculated as offsets from the last position. Each move builds on the previous one. Best for iteratively forming shapes, spirals, zig-zags, or any flowing patterns.
  • Absolute (G90): Coordinates are calculated from a fixed origin on the bed. Each move goes to a specific point. Use this only when you need precise positions that don't depend on the previous move, like moving to a center point or fixed markers.

Example: Absolute vs Relative in Loops

The same loop code produces very different results depending on the active mode:

# Absolute Mode Example Absolute MoveTo x=10 Brepeat 4 MoveTo x+=5 end
# Relative Mode Example Relative MoveTo x=10 Brepeat 4 MoveTo x+=5 end

Output:

# Absolute Mode Output [gcode_macro Absolute!-tester] gcode: G90 G1 X10.000 G1 X15.000 G1 X20.000 G1 X25.000 G1 X30.000 # Relative Mode Output [gcode_macro Relative!-tester] gcode: G91 G1 X10.000 G1 X5.000 G1 X5.000 G1 X5.000 G1 X5.000
Key observation: In absolute mode, x+=5 calculates a new absolute position (10+5=15, then 15+5=20, etc.). In relative mode, it adds 5 to the current position each time, creating equal-sized steps from wherever the previous move ended.

Assignment Operators

Operators allow for precise control during movement:

  • = Direct Assignment
  • += Add to current
  • -= Subtract from current
  • *= Multiply current
  • /= Divide current

1. Static Repeat (repeat)

The repeat command simply duplicates the enclosed G-code commands a specified number of times during compilation.

repeat 3 MoveTo x+=10 y+=10 Respond MSG "Moving..." end

Output (repeat):

G1 X10.0 Y10.0 RESPOND MSG="Moving..." G1 X20.0 Y20.0 RESPOND MSG="Moving..." G1 X30.0 Y30.0 RESPOND MSG="Moving..."

The commands are physically duplicated in the output G-code. Great for simple, fixed repetitions.

2. Dynamic Loop (Brepeat)

The Brepeat command creates a loop that supports arithmetic expressions and mathematical functions. Unlike repeat, this loop exposes an iterator variable i which increments each iteration starting from 0. Expressions are evaluated during compilation, producing numeric G-code values.

Important: The iterator i is only valid inside Brepeat loops, but mathematical functions (sin, cos, tan, sqrt, abs, sign) and the power operator (^) are globally available anywhere an expression is allowed (e.g., in MoveTo, Heat, SetSpeed, etc.).

Example: Using the iterator

Brepeat 5 MoveTo x=i end

Example: Mathematical Expressions (global availability)

Bellerophon supports arithmetic expressions anywhere: including exponentiation ^, absolute value abs(), signum sign(), and the constant pi.

Brepeat 3 MoveTo x=i^2 * 5 + abs(i-2) end Heat extruder=180 + 20*sin(3.14159/2) # works outside loops too

Example: Drawing a Square (Relative)

Using Relative mode ensures each move is from the previous point, naturally forming shapes when iterating.

Relative Brepeat 4 MoveTo x=10 y=0 MoveTo x=0 y=10 MoveTo x=-10 y=0 MoveTo x=0 y=-10 end

Example: Drawing a Circle (Relative)

Using Relative mode allows the circle to be built incrementally, keeping each move relative to the last point for smooth motion.

Relative Brepeat 60 MoveTo x=cos(i*0.1)*5 y=sin(i*0.1)*5 end

Example: Drawing a Spiral (Relative)

Using Relative mode allows complex parametric shapes to be built incrementally. This spiral uses the loop counter i to calculate both angle and radius, creating an expanding spiral pattern.

# Spiral test macro M.title "spiral_test" MoveTo x = 160 y = 160 z= 10 Relative SetSpeed = 6500 Brepeat 65 MoveTo x=cos(i*10)*(i*0.1) y=sin(i*10)*(i*0.1) end Dwell 2 ms Absolute MoveTo x = 150 y = 150 Dwell 2 s Home M.end

How the spiral formula works: The expression cos(i*10)*(i*0.1) and sin(i*10)*(i*0.1) create an expanding spiral where:
  • i*10 controls the angular step (angle increases quickly, making many turns)
  • i*0.1 controls the radius growth (spiral expands outward slowly)
  • The product creates a spiral that grows in radius as it rotates
  • Using Relative mode means each move is an offset from the previous position
