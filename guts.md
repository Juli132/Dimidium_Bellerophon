# Engine Reference – File-by-File Breakdown

This document is a companion to [`CONTRIBUTING.md`](./CONTRIBUTING.md). It provides a **file‑by‑file breakdown** and detailed architecture notes for the Bellerophon Engine.

## Bellerophon Compiler Engine

### Grammar Source

| File | Description |
| :--- | :--- |
| `grammar/jupitore/gen/jupitore.g4` | ANTLR grammar defining the Bellerophon DSL syntax and parse structure. |

> *Note: The core parsing engine, ANTLR grammar, and generated syntax tree files use the legacy project name `Jupitore`. These operate as the core engine for the Bellerophon compiler.*

---

### Generated ANTLR Sources (Build Artifacts)

These files are automatically generated by ANTLR during the Maven build process. They are written to `target/generated-sources/antlr4/` and **should not be edited manually**.

To generate them, run:

```bash
./mvnw clean compile
```

| File | Description |
| :--- | :--- |
| `JupitoreLexer.java` | Lexer that tokenizes Bellerophon source. |
| `JupitoreParser.java` | Parser that builds the parse tree from tokens. |
| `JupitoreVisitor.java` | Visitor interface for parse tree traversal. |
| `JupitoreBaseVisitor.java` | Base visitor with default traversal behavior. |
| `JupitoreListener.java` | Listener interface for parse tree events. |
| `JupitoreBaseListener.java` | Base listener with empty event methods. |

### Compiler Core (`src/main/java/maindeveloper/core/`)

| File | Description |
| :--- | :--- |
| `GCodeVisitor.java` | Core compiler engine. Visits the AST and emits firmware-agnostic motion, macro, and extrusion commands. |
| `Compute.java` | Expression evaluator for arithmetic, vector math, and parameter expansion. |
| `HardwareLimiter.java` | Safety layer that enforces axis limits and rejects invalid motion during compile-time validation. |
| `LayerHandler.java` | Helper for layer-specific macro generation and per-layer state management. |
| `PrinterSettings.java` | Backend model capturing printer, extruder, and temperature settings. |
| `PrinterProfile.java` | Printer profile model for build volume, nozzle characteristics, and motion limits. |
| `WebServer.java` | Spark Java server startup, HTTP routing, compile/highlight endpoints, and static file hosting. |

| File | Description |
| :--- | :--- |
| `KlipperVisitor.java` | Klipper firmware adapter. Implements Bellerophon emission rules for Klipper macros and command syntax. |

## Adapter Architecture

The Bellerophon compiler uses a decoupled firmware adapter architecture:

1. Source files are parsed by the ANTLR grammar into an AST.
2. The compiler engine walks the tree using `GCodeVisitor`.
3. Firmware-specific adapters (`KlipperVisitor`, `MarlinVisitor`) implement the abstract emission hooks.

### Adding a New Firmware Target

| Step | Location | Action |
| :--- | :--- | :--- |
| 1 | `src/main/java/maindeveloper/dialects/` | Create a new Java class (e.g., `RepRapVisitor.java`) that extends `GCodeVisitor`. |
| 2 | `src/main/java/maindeveloper/core/WebServer.java` | Register the new visitor as a compilation target. |
| 3 | `webpage/scripts/ide_scripts/firmware.js` | Add the new firmware option to the target dropdown. |
